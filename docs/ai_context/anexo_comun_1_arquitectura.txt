# ANEXO COMÚN — ARQUITECTURA Y ROADMAP

## VISIÓN DEL SISTEMA

Se desea construir una plataforma astronómica mobile-first basada en Raspberry Pi 5, con control desde PWA, orientada a:
- control de montura SkyWatcher AZ-Go2 WiFi
- control de cámara Player One Mars M
- observación planetaria/lunar avanzada
- EAA/live stacking
- orbital tracking de ISS/LEO
- sesiones, planificación, procesamiento y automatización
- operación offline prioritaria
- simulador desde el inicio

## ARQUITECTURA GENERAL

### Capas
1. Hardware
2. Device abstraction/integration
3. Core backend services
4. Astronomy engine / domain logic
5. Frontend PWA
6. Deployment and ops

### Ejecución física
- Raspberry Pi 5: backend, procesamiento, almacenamiento, simulador, scheduler
- Android Pixel 8a: cliente principal PWA
- Red: IP local sobre USB tethering o WiFi local
- Montura: WiFi
- Cámara: USB 3.0

## DECISIONES CLAVE

- Frontend principal: PWA responsive
- Stack backend: Python + FastAPI
- Frontend: React + TypeScript + Vite + Tailwind
- DB: SQLite en V1
- Offline-first
- Simulador obligatorio de primer nivel
- Seguridad básica de red local en V1
- Acceso remoto solo dentro de red local en V1
- Máscara IA pasa a V1.5 experimental
- Drivers nativos directos primero; INDI solo si hace falta
- Plate solving local prioritario
- La calibración no debe depender continuamente del solve
